## Key Constraints

- Markdown-only store — no vector DB, no embeddings required, plain files readable by any LLM
- PyPI is the primary distribution channel (`pip install chronicle-dev`); npm is secondary
- Extraction is cached by commit SHA — re-running `chronicle init` is always safe and free
- Post-commit hook must be non-blocking (async fire-and-forget)
- `@chronicle/core` must have zero CLI dependencies

## Architecture

- **Monorepo**: `packages/core` (pure logic, no CLI deps) + `packages/cli` (commands, LLM providers, formatters)
- **Store**: `.lore/` directory — `decisions.md`, `rejected.md`, `evolution.md`, `index.md`, `sessions/`, `decisions/` (deep ADRs)
- **Extraction loop**: git history → batch LLM calls → confidence scoring → append to store
- **Inject**: read store → rank by relevance + confidence → trim to token budget → stdout
- **CI loop**: GitHub Actions → `chronicle process` → commit updated `.lore/` → self-maintaining
